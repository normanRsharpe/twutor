The model gateway belongs to platform engineering because every team will otherwise reinvent auth, logging, fallbacks, and cost controls badly.
Receipt: API gateways centralized cross-cutting concerns for services. Model gateways centralize cross-cutting concerns for LLM usage: routing, policy, observability, spend, eval hooks.
AR
AI Reliability Nerd @ship_evals
Careful. If the gateway only logs tokens and latency, it is finance middleware. The useful version captures prompts, retrieved context, tool calls, outputs, and feedback.
Because AI failures are behavioral. You need traces that explain *why* an answer happened, not just that a request took 2.3 seconds.
SS
Security Skeptic @trust_nothing
If your AI platform cannot answer “which tool was called, with what permission, because of which user request?” you do not have governance. You have vibes.
Agent systems need audit trails. Treat tool calls like production operations: scoped credentials, approvals, logs, and replayable incident context.
